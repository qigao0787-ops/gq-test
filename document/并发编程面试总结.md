## 一、并发编程基础

### 1.1 为什么要用并发编程

```
┌─────────────────────────────────────────────────────┐
│              并发编程的优势                          │
├─────────────────────────────────────────────────────┤
│  1. 加快响应时间 - 多线程下载、浏览器多域名加载      │
│  2. 模块化/异步化 - 下单与短信/邮件通知解耦          │
│  3. 充分利用CPU - 多核CPU并行计算                    │
└─────────────────────────────────────────────────────┘
```

### 1.2 线程状态转换

```
        ┌──────────────┐
        │   NEW新建    │
        └──────┬───────┘
               │ start()
               ▼
        ┌──────────────┐
        │ RUNNABLE运行 │◄──────┐
        └──────┬───────┘       │
               │                │
    ┌──────────┼──────────┐    │
    ▼                      ▼    │
┌────────┐            ┌────────┐ │
│BLOCKED │            │WAITING │ │
│ 阻塞   │            │ 等待   │ │
└───┬────┘            └───┬────┘ │
    │                     │      │
    │   synchronized      │  wait()
    │         ◄───────────┘  join()
    │                         │
    ▼                         │
┌──────────────┐              │
│  TERMINATED  │──────────────┘
│    终止      │
└──────────────┘
```

### 1.3 wait() / notify() / sleep()

| 方法 | 所属 | 锁释放 | 用途 |
|------|------|--------|------|
| wait() | Object | ✅ 是 | 线程等待 |
| notify() | Object | ❌ 否 | 唤醒一个等待线程 |
| notifyAll() | Object | ❌ 否 | 唤醒所有等待线程 |
| sleep() | Thread | ❌ 否 | 线程休眠 |

```java
// 生产者-消费者模式
public class ProducerConsumer {
    private int queueSize = 10;
    private Queue<Integer> queue = new LinkedList<>();
    
    public synchronized void put(Integer i) {
        while (queue.size() == queueSize) {
            try { wait(); } catch (InterruptedException e) {}
        }
        queue.add(i);
        notifyAll();  // 唤醒消费者
    }
    
    public synchronized Integer take() {
        while (queue.isEmpty()) {
            try { wait(); } catch (InterruptedException e) {}
        }
        Integer i = queue.remove();
        notifyAll();  // 唤醒生产者
        return i;
    }
}
```

## 二、Future & 异步编程

### 2.1 Callable vs Runnable

```java
// Runnable - 无返回值
@FunctionalInterface
public interface Runnable {
    void run();  // 不能返回值，不能抛异常
}

// Callable - 有返回值
@FunctionalInterface
public interface Callable<V> {
    V call() throws Exception;  // 可返回值，可抛异常
}
```

### 2.2 Future 使用

```java
FutureTask task = new FutureTask<>(() -> {
    Thread.sleep(3000);
    return "任务结果";
});
new Thread(task).start();

// 获取结果（阻塞）
String result = task.get();

// 带超时获取
String result = task.get(5, TimeUnit.SECONDS);
```

### 2.3 Future API

| 方法 | 说明 |
|------|------|
| cancel(boolean) | 取消任务 |
| isCancelled() | 是否已取消 |
| isDone() | 是否已完成 |
| get() | 阻塞获取结果 |
| get(timeout) | 超时获取结果 |

### 2.4 CompletableFuture 实战

```java
// 1. 创建异步任务
CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
    return "Hello";
});

// 2. 串行处理
CompletableFuture<String> result = future.thenApply(s -> s + " World");

// 3. 并行处理
CompletableFuture<String> f1 = CompletableFuture.supplyAsync(() -> "Hello");
CompletableFuture<String> f2 = CompletableFuture.supplyAsync(() -> "World");
CompletableFuture<String> combined = f1.thenCombine(f2, (s1, s2) -> s1 + " " + s2);

// 4. 异常处理
CompletableFuture<String> safe = future.exceptionally(ex -> "Error: " + ex.getMessage());
```

## 三、ThreadLocal

### 3.1 概念

```
ThreadLocal：线程局部变量
- 线程隔离：每个线程独立副本
- 传递数据：同一线程不同组件共享数据
- 常用场景：数据库连接、Session管理、TranceId传递
```

### 3.2 基本使用

```java
public class ThreadLocalDemo {
    private static ThreadLocal<String> tl = new ThreadLocal<>();
    
    public static void main(String[] args) {
        new Thread(() -> {
            tl.set("线程A的数据");
            System.out.println(tl.get());  // 线程A的数据
            tl.remove();
        }).start();
        
        new Thread(() -> {
            tl.set("线程B的数据");
            System.out.println(tl.get());  // 线程B的数据
            tl.remove();
        }).start();
    }
}
```

### 3.3 内存泄漏问题

```
┌─────────────────────────────────────────────────────┐
│              ThreadLocal 内存泄漏                    │
├─────────────────────────────────────────────────────┤
│  ThreadLocalMap Entry 结构：                         │
│  static class Entry extends WeakReference<ThreadLocal<?>> { │
│      Object value;  // 强引用                        │
│  }                                                  │
│                                                     │
│  问题：value是强引用，ThreadLocal回收后，key为null   │
│       但value无法回收，导致内存泄漏                  │
│                                                     │
│  解决：手动调用 remove()                            │
└─────────────────────────────────────────────────────┘
```

### 3.4 最佳实践

```java
// 一定要remove！
try {
    tl.set(userId);
    // 业务逻辑
} finally {
    tl.remove();  // 必须！
}

// 或使用 initialValue
private static ThreadLocal<User> userHolder = 
    ThreadLocal.withInitial(() -> new User());
```

## 四、CAS & Atomic

### 4.1 CAS 原理

```
Compare And Swap：比较并交换

CAS(V, Expected, NewValue):
    if V == Expected:
        V = NewValue
        return true
    else:
        return false

┌─────────────────────────────────────┐
│           CAS 操作流程               │
├─────────────────────────────────────┤
│  1. 读取当前值 A                     │
│  2. 计算新值 B                       │
│  3. CAS(A, B) 比较                   │
│  4. 成功则更新，失败重试              │
└─────────────────────────────────────┘
```

### 4.2 CAS 存在的问题

| 问题 | 说明 |
|------|------|
| ABA问题 | 变量从A→B→A，CAS无法检测 |
| 自旋开销 | 失败循环，CPU开销大 |
| 只能保证单变量 | 多变量无法保证原子性 |

```java
// ABA问题解决 - 使用版本号
AtomicStampedReference<Integer> ref = new AtomicStampedReference<>(100, 1);
ref.compareAndSet(100, 101, 1, 2);
ref.compareAndSet(101, 100, 2, 3);
```

### 4.3 Atomic 原子类

```java
// 原子更新基本类型
AtomicInteger atomicInt = new AtomicInteger(0);
atomicInt.incrementAndGet();  // ++i
atomicInt.decrementAndGet();  // --i
atomicInt.getAndAdd(5);       // i += 5
atomicInt.compareAndSet(0, 10);  // CAS

// 原子更新数组
AtomicIntegerArray array = new AtomicIntegerArray(new int[]{1,2,3});
array.getAndSet(0, 10);

// 原子更新对象
AtomicReference<User> userRef = new AtomicReference<>();
userRef.compareAndSet(oldUser, newUser);

// 原子更新字段
AtomicIntegerFieldUpdater<User> updater = 
    AtomicIntegerFieldUpdater.newUpdater(User.class, "age");
updater.incrementAndGet(user);
```

## 五、synchronized 原理

### 5.1 锁的升级过程

```
┌─────────────────────────────────────────────────────┐
│           synchronized 锁升级                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│   无锁    ──►  偏向锁   ──►   轻量级锁   ──►  重量级锁  │
│                                                       │
│   01        01           00              10          │
│                                                     │
│   首次       只有一个       多线程          竞争激烈   │
│   进入       线程进入       交替执行         阻塞等待   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 5.2 锁的状态

| 状态 | 存储位置 | 特点 |
|------|----------|------|
| 无锁 | 对象头 | 初始状态 |
| 偏向锁 | 对象头 | 第一次获取，MarkWord记录线程ID |
| 轻量级锁 | 栈帧 | CAS自旋，线程栈帧存储锁记录 |
| 重量级锁 | 对象头 | MonitorObject，阻塞唤醒 |

### 5.3 核心汇编指令

```java
// synchronized代码块编译后会生成 monitorenter / monitorexit
public void sync() {
    synchronized(this) {  // monitorenter
        // 临界区
    }  // monitorexit
}
```

### 5.4 synchronized 关键字用法

```java
// 1. 修饰实例方法 - 锁this
public synchronized void method() { }

// 2. 修饰静态方法 - 锁类对象
public synchronized static void staticMethod() { }

// 3. 修饰代码块 - 锁指定对象
public void block() {
    synchronized(this) { }
    synchronized(lock对象) { }
    synchronized(类.class) { }
}
```

## 六、AQS & ReentrantLock

### 6.1 AQS 核心结构

```java
// AbstractQueuedSynchronizer
public abstract class AbstractQueuedSynchronizer {
    // 头结点（当前持有锁的线程）
    private transient volatile Node head;
    // 尾结点（等待队列）
    private transient volatile Node tail;
    // 同步状态
    private volatile int state;
    
    // Node结构
    static class Node {
        volatile int waitStatus;  // 等待状态
        volatile Node prev;       // 前驱
        volatile Node next;       // 后继
        Thread thread;            // 等待线程
    }
}
```

### 6.2 AQS 两种模式

| 模式 | 说明 | 方法 |
|------|------|------|
| 独占锁 | 一次只有一个线程获取锁 | acquire() / release() |
| 共享锁 | 多个线程同时获取 | acquireShared() / releaseShared() |

### 6.3 ReentrantLock 使用

```java
// 可重入锁
private ReentrantLock lock = new ReentrantLock();

public void method() {
    lock.lock();
    try {
        // 临界区
    } finally {
        lock.unlock();  // 必须finally中释放
    }
}

// 尝试获取锁
if (lock.tryLock()) {
    try {
        // 获取到锁
    } finally {
        lock.unlock();
    }
}

// 带超时获取
if (lock.tryLock(5, TimeUnit.SECONDS)) {
    try { } finally { lock.unlock(); }
}
```

### 6.4 ReentrantLock vs synchronized

| 特性 | ReentrantLock | synchronized |
|------|---------------|--------------|
| 锁实现 | Java代码 | JVM实现 |
| 公平锁 | 支持 | 不支持 |
| 尝试非阻塞 | tryLock() | ❌ |
| 超时获取 | tryLock(timeout) | ❌ |
| 条件变量 | newCondition() | wait/notify |
| 可中断 | lockInterruptibly() | ❌ |

## 七、JUC 工具类

### 7.1 Semaphore（信号量）

```java
// 限流：同时只有N个线程执行
Semaphore semaphore = new Semaphore(3);

public void doSomething() {
    try {
        semaphore.acquire();  // 获取许可证
        // 临界区
    } finally {
        semaphore.release();  // 释放许可证
    }
}
```

### 7.2 CountDownLatch（倒计时门闩）

```java
// 等待N个线程完成
CountDownLatch latch = new CountDownLatch(5);

for (int i = 0; i < 5; i++) {
    new Thread(() -> {
        // 执行任务
        latch.countDown();  // 计数-1
    }).start();
}

latch.await();  // 等待所有线程完成
System.out.println("所有任务完成");
```

### 7.3 CyclicBarrier（循环栅栏）

```java
// 等待N个线程到达后再继续
CyclicBarrier barrier = new CyclicBarrier(3);

for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        // 执行任务
        barrier.await();  // 等待其他线程
        System.out.println("继续执行");
    }).start();
}
```

### 7.4 对比

| 工具 | 特点 | 能否复用 |
|------|------|----------|
| CountDownLatch | 倒计时，一次性 | 不可 |
| CyclicBarrier | 循环栅栏，可复用 | 可 |
| Semaphore | 信号量，限流 | 可 |

            // 小任务直接计算

        RecursiveTask<Long> left = new RecursiveTask<>() {

### 10.2 三大特性

| 特性 | 说明 | 解决方式 |
|------|------|----------|
| 原子性 | 操作的不可分割性 | synchronized, Lock |
| 可见性 | 线程间可见 | volatile, synchronized |
| 有序性 | 指令重排序 | volatile, synchronized |

### 10.3 volatile 关键字

```java
// 保证可见性和有序性，不保证原子性
private volatile boolean flag = false;

// 双重检查锁定单例
public class Singleton {
    private static volatile Singleton instance;
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

### 10.4 happens-before 原则

```
程序顺序规则：一个线程中每个操作happens-before该线程中后续操作
监视器锁规则：解锁happens-before后续加锁
volatile规则：volatile写happens-before后续读取
传递性：A happens-before B, B happens-before C → A happens-before C
```

## 十一、Disruptor 高性能队列

### 11.1 核心概念

```
Disruptor：高性能队列
- 特点：无锁、高并发、RingBuffer
- 性能：单机可达600万/秒
- 适用：生产者-消费者模式、高频交易
```

### 11.2 核心组件

```java
// 1. 定义事件
public class LongEvent {
    private long value;
    // getter/setter
}

// 2. 定义工厂
public class LongEventFactory implements EventFactory<LongEvent> {
    public LongEvent newInstance() { return new LongEvent(); }
}

// 3. 定义消费者
public class LongEventHandler implements EventHandler<LongEvent> {
    public void onEvent(LongEvent event, long sequence, boolean endOfBatch) {
        System.out.println("Event: " + event.getValue());
    }
}

// 4. 构建Disruptor
Disruptor<LongEvent> disruptor = new Disruptor<>(
    new LongEventFactory(),
    1024 * 1024,  // RingBuffer大小
    new ThreadPoolExecutor(5, 5, 0, TimeUnit.MILLIS,
        new ArrayBlockingQueue<>(100)),
    ProducerType.MULTI,
    new BlockingWaitStrategy()
);

disruptor.handleEventsWith(new LongEventHandler());
disruptor.start();

// 5. 生产消息
RingBuffer<LongEvent> ringBuffer = disruptor.getRingBuffer();
long sequence = ringBuffer.next();
try {
    LongEvent event = ringBuffer.get(sequence);
    event.setValue(999L);
} finally {
    ringBuffer.publish(sequence);
}
```

### 11.3 为什么快

```
1. 环形数组 - 预分配内存，无GC
2. 元素位置定位 - 位运算，O(1)
3. 无锁设计 - CAS操作
4. 缓存行填充 - 避免伪共享
5. 批量处理 - 减少上下文切换
```
