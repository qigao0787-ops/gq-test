## 一、Spring Boot

### 1.1 自动装配原理

```java
// @SpringBootApplication = @SpringBootConfiguration
//                          + @EnableAutoConfiguration
//                          + @ComponentScan

// 1. @EnableAutoConfiguration
@EnableAutoConfiguration
    ↓
AutoConfigurationImportSelector
    ↓
getCandidateConfigurations()
    ↓
META-INF/spring.factories
META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports

// 2. 条件装配
@ConditionalOnClass(DataSource.class)
@ConditionalOnProperty(prefix="spring.datasource", name="url")
```

### 1.2 启动流程

```
SpringApplication.run()
    │
    ├─> 1. 创建 ApplicationContext
    │
    ├─> 2. 加载 META-INF/spring.factories
    │
    ├─> 3. SpringApplicationRunListeners.started()
    │
    ├─> 4. prepareEnvironment()
    │       └─> 加载 application.yml
    │
    ├─> 5. prepareContext()
    │       └─> 扫描 @ComponentScan
    │
    ├─> 6. refreshContext()
    │       └─> 启动内嵌 Tomcat
    │
    └─> 7. afterRefresh()
            └─> 执行 Runner
```

### 1.3 Starter 原理

```java
// 1. 定义 Starter
// my-starter-autoconfigure
//   - 配置类：MyAutoConfiguration
//   - META-INF/spring.factories

// 2. 使用 Starter
// my-starter
//   - 引入 autoconfigure
//   - 无任何代码
```

### 2.2 Nacos

```java
  cloud:
```

### 2.3 Sentinel

```java
// 1. 流控规则
// - 直接拒绝
// - 冷启动 (Warm Up)
```

### 2.4 Seata 分布式事务

```
┌─────────────────────────────────────────────────────┐
│              Seata AT 模式                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│   TC (Transaction Coordinator) - Server            │
│       │                                            │
│   TM (Transaction Manager) - 发起方                 │
│       │                                            │
│   RM (Resource Manager) - 参与方                    │
│                                                     │
│   流程：                                            │
│   1. TM 向 TC 注册全局事务                          │
│   2. RM 向 TC 注册分支事务                          │
│   3. 各分支执行，保存 undo_log                      │
│   4. TC 汇总结果，下发 commit/rollback              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

```java
// 使用 Seata
@GlobalTransactional
public void createOrder() {
    // 扣库存
    storageService.decrease(productId, count);
    // 扣余额
    accountService.decrease(userId, money);
    // 创建订单
    orderService.create(order);
}
```

## 三、微服务设计

### 3.1 拆分原则

```
┌─────────────────────────────────────────────────────┐
│            微服务拆分原则                            │
├─────────────────────────────────────────────────────┤
│  1. 单一职责 - 一个服务一个业务域                     │
│  2. 自治独立 - 独立部署、数据库、团队                 │
│  3. 粒度适中 - 避免过细/过粗                          │
│  4. 演进式 - 逐步拆分                                │
└─────────────────────────────────────────────────────┘
```

### 3.2 通信方式

| 方式 | 特点 | 场景 |
|------|------|------|
| HTTP/REST | 简单、通用 | 同步调用 |
| RPC | 高性能 | 内部服务 |
| 消息队列 | 异步、解耦 | 耗时操作 |

### 3.3 服务治理

```
服务注册/发现 → 负载均衡 → 容错 → 网关
      ↓            ↓          ↓        ↓
    Nacos       Ribbon     Sentinel  Gateway

配置管理 → 链路追踪 → 日志监控
    ↓           ↓          ↓
  Nacos       Sleuth     ELK
```
