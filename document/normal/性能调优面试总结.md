## 一、JVM

### 1.1 内存区域

```
┌─────────────────────────────────────────────────────┐
│                 JVM 内存区域                         │
├─────────────────────────────────────────────────────┤
│                                                     │
│  堆 (Heap) - 线程共享                               │
│  ┌─────────────────────────────────────────────┐   │
│  │  Young Generation (Eden + 2 Survivor)       │   │
│  │    ├── Eden                                 │   │
│  │    ├── S0 (From)                            │   │
│  │    └── S1 (To)                              │   │
│  │                                             │   │
│  │  Old Generation (Tenured)                   │   │
│  └─────────────────────────────────────────────┘   │
│                                                     │
│  元空间 (Metaspace) - 类信息                        │
│                                                     │
│  虚拟机栈 (VM Stack) - 线程私有                     │
│    Stack Frame: 局部变量表/操作数栈/动态链接/返回地址 │
│                                                     │
│  本地方法栈 (Native Stack)                          │
│                                                     │
│  程序计数器 (PC Register)                           │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 1.2 对象创建过程

```java
// 1. 类加载检查
Class<?> com.example.User

// 2. 分配内存
// - 指针碰撞 (Serial/ParNew)
// - 空闲列表 (CMS)

// 3. 初始化零值
object header + zero fields

// 4. 设置对象头
Mark Word: hashCode + age + bias + lock

// 5. <init> 方法
执行构造函数
```

### 1.3 垃圾回收算法

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 标记-清除 | 标记存活对象，清除 | 简单 | 内存碎片 |
| 标记-整理 | 标记+整理 | 无碎片 | 效率低 |
| 复制 | Eden→S0→S1 | 高效 | 内存减半 |
| 分代收集 | 分代策略 | 最优 | 需要适配 |

### 1.4 垃圾收集器

```
┌─────────────────────────────────────────────────────┐
│            垃圾收集器选择                            │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Young:                                             │
│  - Serial (单线程) - client                         │
│  - ParNew (并行) - CMS配合                          │
│  - Parallel Scavenge (吞吐量优先)                   │
│                                                     │
│  Old:                                               │
│  - Serial Old                                       │
│  - Parallel Old                                     │
│  - CMS (并发标记清除)                               │
│  - G1 (Region)                                      │
│  - ZGC (着色指针)                                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 1.5 G1 收集器

```java
// G1 把堆划分成多个大小相等的 Region
// - Eden Region
// - Survivor Region  
// - Old Region
// - Humongous Region (大对象)

// 收集过程
// 1. Young GC (年轻代收集)
// 2. Mixed GC (混合收集)
// 3. Full GC (全局GC)

// 参数
-XX:MaxGCPauseMillis=200  // 最大停顿时间
-XX:G1HeapRegionSize=8m   // Region大小
-XX:InitiatingHeapOccupancyPercent=45  // 触发GC阈值
```

### 1.6 类加载机制

```
加载 → 验证 → 准备 → 解析 → 初始化 → 使用 → 卸载

加载阶段:
1. 通过类的全限定名获取二进制字节流
2. 将字节流转化为方法区的运行时数据结构
3. 在堆中生成java.lang.Class对象

双亲委派:
  AppClassLoader
       ↑
   ExtClassLoader
       ↑
   BootstrapClassLoader (C++)

加载顺序: 父类静态 → 子类静态 → 父类构造 → 子类构造
```

### 1.7 JVM 调优参数

```bash
# 堆大小
-Xms512m -Xmx512m          # 初始/最大堆
-XX:NewRatio=2             # old/young = 2:1
-XX:SurvivorRatio=8        # Eden/Survivor = 8:1

# 垃圾收集器
-XX:+UseSerialGC           # Serial + Serial Old
-XX:+UseParNewGC           # ParNew + CMS
-XX:+UseParallelGC         # Parallel Scavenge + Parallel Old
-XX:+UseG1GC               # G1
-XX:+UseZGC                # ZGC

# G1
-XX:MaxGCPauseMillis=200
-XX:G1HeapRegionSize=16m

# OOM dump
-XX:+HeapDumpOnOutOfMemoryError
-XX:HeapDumpPath=/tmp/heap.hprof
```

### 1.8 调优工具

```bash
# 命令行
jps -l                      # 进程列表
jinfo -flags <pid>         # JVM参数
jstat -gc <pid> 1000       # GC统计
jstack <pid>               # 线程堆栈
jmap -heap <pid>           # 堆信息
jmap -dump:format=b,file=heap.hprof <pid>

# 可视化
jconsole
jvisualvm
Arthas
```

### 2.2 InnoDB 存储引擎

```java
// 1. 数据组织：B+Tree 索引
// 2. 事务支持：ACID
// 3. 行级锁
// 4. MVCC

// 索引结构
B+Tree (非叶子节点不存储数据)
  │
  ├── 主键索引 (聚簇索引)
  │   Leaf: 主键值 + 完整行数据
  │
  └── 辅助索引 (非聚簇索引)
      Leaf: 索引列值 + 主键值
```

### 2.3 索引优化

```sql
-- 索引选择原则
-- 1.where条件字段
-- 2.区分度高的列
-- 3.尽量使用覆盖索引
-- 4.避免索引失效

-- 索引失效场景
SELECT * FROM t WHERE name LIKE '%abc'  -- 前导通配符
SELECT * FROM t WHERE SUBSTR(name,1,3)='abc'  -- 函数
SELECT * FROM t WHERE id + 1 = 10  -- 运算
SELECT * FROM t WHERE age = 18 OR score = 100  -- OR
SELECT * FROM t ORDER BY name  -- 跨列排序

-- 最佳实践
-- 1. 覆盖索引
SELECT id, name FROM t WHERE name='Tom'  -- 不回表

-- 2. 最左前缀
-- (a,b,c) 索引可用: a / a,b / a,b,c

-- 3. 索引下推
-- 减少回表次数
```

### 2.4 Explain 分析

```sql
EXPLAIN SELECT * FROM t WHERE id = 1

-- 关键字段
type: 访问类型 (system > const > eq_ref > ref > range > index > ALL)
key: 实际使用索引
rows: 扫描行数
extra: Using filesort / Using index / Using where
```

### 2.5 事务隔离级别

| 级别 | 脏读 | 不可重复读 | 幻读 |
|------|------|-----------|------|
| READ UNCOMMITTED | √ | √ | √ |
| READ COMMITTED | × | √ | √ |
| REPEATABLE READ (默认) | × | × | √ |
| SERIALIZABLE | × | × | × |

```sql
-- 设置隔离级别
SET transaction isolation level READ COMMITTED

-- MVCC (多版本并发控制)
-- 隐藏字段: row_id, transaction_id, roll_pointer
-- undo log: 记录历史版本
-- read_view: 可见性判断
```

### 2.6 锁机制

```sql
-- 1. 行锁
SELECT * FROM t WHERE id = 1 LOCK IN SHARE MODE  -- 共享锁
SELECT * FROM t WHERE id = 1 FOR UPDATE           -- 排他锁

-- 2. 间隙锁 (RR级别)
SELECT * FROM t WHERE id > 10 AND id < 20  
-- 锁定 (10,20) 区间，防止幻读

-- 3. 死锁
-- 解决: 多个事务按相同顺序获取锁
-- 检测: innodb_lock_wait_timeout
```

### 2.7 SQL 优化

```sql
-- 1. 分页优化
-- 慢: SELECT * FROM t LIMIT 1000000, 10
-- 快: SELECT * FROM t WHERE id > 1000000 LIMIT 10

-- 2. JOIN优化
-- 小表驱动大表
-- 建议: 被驱动表有索引

-- 3. 避免 SELECT *
-- 只查询需要的字段

-- 4. 批量插入
INSERT INTO t VALUES (1,'a'),(2,'b'),(3,'c')
```
