## 一、设计原则 (SOLID)

| 原则 | 说明 | 一句话 |
|------|------|--------|
| S - 单一职责 | 一个类只做一件事 | 一个类一个变化原因 |
| O - 开闭原则 | 对扩展开放,对修改关闭 | 加功能不改原代码 |
| L - 里氏替换 | 子类可替换父类 | 子类不破坏父类行为 |
| I - 接口隔离 | 接口要小而专 | 不强迫实现不需要的方法 |
| D - 依赖倒置 | 依赖抽象不依赖具体 | 面向接口编程 |

## 二、创建型模式

### 2.1 单例模式 ★★★★★

```java
// 1. 双重检查锁 (DCL)
public class Singleton {
    private static volatile Singleton instance;  // volatile 防止指令重排
    private Singleton() {}
    
    public static Singleton getInstance() {
        if (instance == null) {                 // 第一次检查(避免不必要的锁)
            synchronized (Singleton.class) {
                if (instance == null) {         // 第二次检查(防止重复创建)
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

// 2. 静态内部类 (推荐)
public class Singleton {
    private Singleton() {}
    private static class Holder {
        static final Singleton INSTANCE = new Singleton();
    }
    public static Singleton getInstance() {
        return Holder.INSTANCE;  // 类加载时初始化,天然线程安全
    }
}

// 3. 枚举 (最安全,防反射/序列化破坏)
public enum Singleton {
    INSTANCE;
    public void doSomething() {}
}
```

**Spring 中的单例：** `@Scope("singleton")` 默认，Bean 存在三级缓存中。

### 2.2 工厂模式 ★★★★

```java
// 简单工厂
public class PayFactory {
    public static Pay create(String type) {
        switch (type) {
            case "alipay": return new AliPay();
            case "wechat": return new WechatPay();
            default: throw new IllegalArgumentException();
        }
    }
}

// 抽象工厂
public interface PayFactory {
    Pay createPay();
    Refund createRefund();
}
public class AliPayFactory implements PayFactory {
    public Pay createPay() { return new AliPay(); }
    public Refund createRefund() { return new AliRefund(); }
}
```

**Spring 中：** `BeanFactory` / `ApplicationContext` 就是工厂模式。

### 2.3 建造者模式 ★★★

```java
// Lombok @Builder 自动生成
User user = User.builder()
    .name("Tom")
    .age(25)
    .email("tom@example.com")
    .build();

// 适用: 参数多、可选参数多的对象创建
// Spring 中: StringBuilder / Stream.Builder / BeanDefinitionBuilder
```

## 三、结构型模式

### 3.1 代理模式 ★★★★★

```
┌─────────────────────────────────────────────────────┐
│              代理模式                                │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Client → Proxy → RealSubject                       │
│                                                     │
│  静态代理: 编译期确定代理类                          │
│  动态代理: 运行时生成代理类                          │
│    - JDK: 基于接口 (Proxy + InvocationHandler)      │
│    - CGLIB: 基于继承 (Enhancer + MethodInterceptor) │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**Spring AOP 就是代理模式：**
- 有接口 → JDK 动态代理
- 无接口 → CGLIB
- `@Transactional` / `@Async` / `@Cacheable` 底层都是代理

### 3.2 适配器模式 ★★★

```java
// 将不兼容的接口转换为目标接口
// Spring MVC: HandlerAdapter 适配不同类型的 Controller
// - RequestMappingHandlerAdapter (@RequestMapping)
// - SimpleControllerHandlerAdapter (Controller 接口)
// - HttpRequestHandlerAdapter (HttpRequestHandler)

public interface HandlerAdapter {
    boolean supports(Object handler);
    ModelAndView handle(HttpServletRequest request, 
                       HttpServletResponse response, Object handler);
}
```

### 3.3 装饰器模式 ★★★

```java
// 动态为对象添加功能,不改变原类
// Java IO 就是典型装饰器:
InputStream is = new BufferedInputStream(    // 装饰: 加缓冲
                    new FileInputStream("file.txt"));  // 被装饰

// Spring: BeanDefinitionDecorator / TransactionAwareCacheDecorator
```

### 3.4 门面模式(外观) ★★★

```java
// 提供统一的高层接口,屏蔽子系统复杂性
// Spring: JdbcTemplate (封装了 Connection/Statement/ResultSet)
// SLF4J (统一日志门面)

public class OrderFacade {
    public void createOrder(OrderDTO dto) {
        inventoryService.deduct(dto);    // 库存
```

## 四、行为型模式

### 4.1 策略模式 ★★★★★

```java
// 定义一系列算法,使它们可以互相替换

// 策略接口
public interface DiscountStrategy {
    BigDecimal calculate(BigDecimal price);
}

// 具体策略
public class VipDiscount implements DiscountStrategy {
    public BigDecimal calculate(BigDecimal price) {
        return price.multiply(new BigDecimal("0.8"));
    }
}

// 上下文 (消除 if-else)
public class PriceService {
    private Map<String, DiscountStrategy> strategyMap;  // Spring 注入
    
    public BigDecimal getPrice(String userType, BigDecimal price) {
        return strategyMap.get(userType).calculate(price);
    }
}
```

**Spring 中：** `Resource` 接口的不同实现(ClassPathResource/FileSystemResource)

### 4.2 观察者模式 ★★★★

```java
// 一对多依赖,主题变化时通知所有观察者

// Spring 事件机制:
// 1. 定义事件
public class OrderCreatedEvent extends ApplicationEvent {
    private Order order;
    public OrderCreatedEvent(Object source, Order order) {
        super(source);
        this.order = order;
    }
}

// 2. 发布事件
applicationEventPublisher.publishEvent(new OrderCreatedEvent(this, order));

// 3. 监听事件
@EventListener
public void onOrderCreated(OrderCreatedEvent event) {
    // 发短信/加积分/...
}

// 好处: 解耦,发布者不需要知道有哪些监听者
```

### 4.3 模板方法模式 ★★★★

```java
// 父类定义算法骨架,子类实现具体步骤

public abstract class AbstractExporter {
    // 模板方法 (final 防止子类覆盖骨架)
    public final void export(List<Data> data) {
        validate(data);        // 固定步骤
        List<Row> rows = convert(data);  // 子类实现
        write(rows);           // 子类实现
        afterExport();         // 钩子方法(可选覆盖)
    }
    
    protected abstract List<Row> convert(List<Data> data);
    protected abstract void write(List<Row> rows);
    protected void afterExport() {}  // 钩子
}

// Spring 中:
// - JdbcTemplate (模板方法 + 回调)
// - AbstractApplicationContext.refresh() (容器启动流程)
// - RestTemplate / RedisTemplate
```

### 4.4 责任链模式 ★★★★

```java
// 请求沿链传递,每个节点决定处理或传递

// Spring 中:
// - Filter Chain (Servlet 过滤器链)
// - Interceptor Chain (HandlerInterceptor)
// - Spring Security FilterChainProxy

public interface Handler {
    void handle(Request request, Handler next);
}

// Netty 中: ChannelPipeline 就是责任链
// pipeline.addLast(decoder, businessHandler, encoder);
```

### 4.5 迭代器模式 ★★

```java
// Java 集合都实现了 Iterable 接口
for (String s : list) { }  // 语法糖,底层就是 Iterator

// 增强 for 编译后:
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
}
```
